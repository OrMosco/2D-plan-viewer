# Product Requirements Document (PRD): AI Structural Analysis Feature

## 1. Context & Objective
Spacial.io is a modern 2D architectural plan viewer (React 18 + TypeScript + Vite). It currently features cursor-centered zoom, panning, a real-time action log, and light/dark modes. 
The objective is to introduce a new feature: **"AI Structural Optimization"**. This will allow the user to click a button, send the currently viewed 2D foundation plan to the Gemini API, and receive both a textual structural analysis and an annotated image with white-pen structural sketches.

## 2. UI/UX Requirements
* **Header Integration:** Add a highly visible, styled button in the main application Header (e.g., `✨ AI Analyze Structure`). It should match the existing modern blue/white UI with smooth hover effects.
* **Loading State:** When the button is clicked, trigger a visually distinct loading state (e.g., a spinner on the button, or a pulsing overlay on the plan viewer) since the AI generation can take a few seconds. Disable the button during fetching.
* **Result Display (Image):** Once the `annotated_image_output` (Base64 string) is returned, seamlessly swap or overlay the original plan image in the main viewer with the new AI-annotated image. The existing zoom and pan functions must continue to work perfectly on this new image.
* **Result Display (Text):** Display the `structural_analysis_summary` in a newly created side panel, modal, or expandable bottom sheet. The text should be cleanly formatted using Markdown rendering if possible.
* **Action Log:** Integrate with the existing real-time action log. 
    * Log: "🤖 AI Structural Analysis requested..."
    * Log: "✅ AI Analysis complete and applied."

## 3. Technical Requirements
* **API Utility:** Create a new utility file (e.g., `src/services/geminiService.ts`) to handle the API call. 
* **Payload Construction:** The utility must take the source image (converted to a Base64 string) and the text prompt, and structure them into the correct JSON payload for the Gemini multimodal endpoint.
* **Environment Variables:** Ensure the Google Gemini API key is securely accessed via Vite's environment variables (`import.meta.env.VITE_GEMINI_API_KEY`).
* **State Management:** Update the main App state to track:
    * `isAnalyzing` (boolean)
    * `aiAnalysisText` (string | null)
    * `currentPlanImage` (string - to allow swapping between the original and annotated Base64 images).

## 4. Specific Prompt to Send to API
Use the following text prompt in the API request:
> "Take the role of a highly qualified structural engineer. It is your task to analyze the attached building foundation plan and propose innovative structural optimizations. Analyze the attached photograph, and list all the advanced structural solutions that make sense for this building. Really go all out, don't miss any. Be extremely creative and innovative. Focus strictly on structural enhancements that will directly benefit the building's stability, load-bearing capacity, and longevity—think of benefits such as seismic resilience, optimized load distribution, differential settlement mitigation, material efficiency, vibration control, and advanced foundational support systems. In white pen, hand-sketch as many of these structural solutions as you can think of directly onto the photograph (while making sure that they are technically sound and practical). Integrate the solutions in elegant, subtle, beautiful ways - really make them shine. Make your sketches very minimal but very beautiful. Annotate these solutions in a neat minimalist font. Leave the photograph as it is, just add the sketches and annotations to it. Do not redraw any part of the photograph. The result must be very beautiful, minimalist, and clear."

## 5. Implementation Steps Requested
1. Provide the code for the `geminiService.ts` to handle the API request and Base64 handling.
2. Provide the updated `Header.tsx` (or equivalent) to include the new button and loading state.
3. Provide the necessary state updates in the main `App.tsx` (or parent component) to handle image swapping and displaying the text summary.
